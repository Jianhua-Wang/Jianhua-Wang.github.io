Transport large file between servers

Sometimes we need to back up data from a server to another using scp or rsync. It is very time-consuming when the data is very large, for example, over 500G. I found a combined command that helped me a lot.

Steps

1. Split your file into small pieces

-b for bytes and -n for number of lines.

2. Transport the small pieces using scp

or you can transport multiple files in a meanwhile. I prefer using multiple threading from Python.

3. Merge the small pieces to the original file

Finished. You can check the MD5 value and even the tabix index is still working.

PS

In my case, speed of transporting once a file is 2M/s, while it’s around 10M/s when I scp 8 files simultaneously. However, you should add the public key to the remote server, (see here for detail), and nohup will block up the processes.
